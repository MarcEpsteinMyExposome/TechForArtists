# IT2-06: HTML Signature Generation

**Status:** PENDING
**Depends on:** IT2-01
**Wave:** 2

## Scope

Pure logic that converts a `Signature` object into email-compatible HTML. Table-based layout, all inline styles, no CSS classes, web-safe fonts.

Files:
- `lib/signature/generateHtml.ts` — main: Signature -> HTML string
- `lib/signature/layouts/horizontal.ts` — horizontal table layout
- `lib/signature/layouts/stacked.ts` — stacked table layout
- `lib/signature/layouts/compact.ts` — compact table layout
- `lib/signature/socialIcons.ts` — social link `<a>` tags (text labels for MVP)
- `lib/signature/utils.ts` — escapeHtml, resolveColorPreset, stripProtocol

Constraints:
- `<table>` elements for layout (no flexbox/grid)
- Only inline `style=""` attributes
- Web-safe fonts: `Arial, Helvetica, sans-serif`
- Skip empty/undefined fields entirely
- Website: display domain only, link to full URL
- HTML entities properly escaped

## Files

**Create:** 6 logic files
**Tests:** `__tests__/lib/signature/generateHtml.test.ts`, `__tests__/lib/signature/utils.test.ts`

## Acceptance

- Each layout generates valid HTML with `<table>` and inline styles
- Colors from resolved preset appear in output
- Empty fields are omitted
- Social links generate correct `<a>` tags
- HTML escaping works (e.g., `&` -> `&amp;`)
- All tests pass
